# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные
#### IProduct
Интерфейс для описания товара.

`id: string` - уникальный идентификатор товара.
`description: string` - подробное описание товара.
`image: string` - URL изображения товара.
`title: string` - название товара.
`category: string` - категория товара.
`price: number | null` - цена товара.

#### IBuyer
Интерфейс для описания покупателя

`payment: 'cash' | 'card' | ''` - способ оплаты.
`email: string` - адрес электронной почты покупателя.
`phone: string` - номер телефона покупателя.
`address: string` - адрес покупателя.

#### IOrder
Интерфейс для описания заказа

`payment: 'cash' | 'card' | ''` - способ оплаты (наследует свойство от IBuyer).
`email: string` - адрес электронной почты покупателя (наследует свойство от IBuyer).
`phone: string` - номер телефона покупателя (наследует свойство от IBuyer).
`address: string` - адрес покупателя (наследует свойство от IBuyer).
`total: number` - сумма заказа.
`items: string[]` - массив товаров в заказе.

### Модели данных

#### Класс Catalog
Представляет собой каталог товаров интернет-магазина WebLarek.

Конструктор: 
`items: IProduct[], selectedItem: IProduct | null` - в конструктор передается массив товаров представленных в интернет-магазине, а также выбранный товар для подробного отображения.

Поля класса:  
`items: IProduct[]` -  хранит массив всех товаров.
`selectedItem: IProduct | null`- хранит ссылку на выбранный товар, используется для отображения подробной информации.

Методы класса:  
`setItems(items: IProduct[]): void` - сохранение массива товаров полученного в параметрах метода.
`getItems():  IProduct[]` - возвращает массив всех товаров, хранящихся в модели.
`getItemId(id: string)` - находит и возвращает товар по его уникальному id.
`setSelectedItem(selectedItem: IProduct | null)` - устанавливает товар как выбранный для последующего подробного отображения.
`getSelectedItem()` - возвращает товар для подробного отображения.

#### Класс Basket
Представляет собой модель корзины покупок для интернет-магазина WebLarek. Обеспечивает функциональность проверки наличия, добавления и удаления товара, очистки корзины, получения количества товара и общей стоимости корзины.

Конструктор:
`events: IEvents` - экземпляр системы событий для обработки взаимодействия.

Поля класса:  
`items: IProduct[]` - хранит массив товаров, выбранных покупателем для покупки.

Методы класса:  
`getItems(): IProduct[]` - получение массива товаров, которые находятся в корзине.
`addItem(item: IProduct)` - добавление товара, который был получен в параметре, в массив корзины.
`deleteItem(id: string)` - удаление товара, полученного в параметре из массива корзины.
`clearItems(): IProduct[]` - очистка корзины.
`getTotalItems(): number` - получение стоимости всех товаров в корзине.
`getCountItems(): number` - получение количества товаров в корзине.
`hasItem(id: string): boolean` - проверка наличия товара в корзине по его уникальному id.

#### Класс Customers
Предназначен для хранения и управления данными покупателя.

Конструктор:
`payment: 'cash' | 'card' | '', email: string, phone: string, address: string` - в конструктор передаются данные покупателя.

Поля класса:  
`payment: 'cash' | 'card' | ''` - хранит данные о способе оплаты.
`email: string` - хранит данные электронной почты покупателя.
`phone: string` - хранит данные номера телефона покупателя
`address: string` - хранит данные адреса покупателя.

Методы класса:
`setPayment(payment: TPayment): void` - хранение данных покупателя о выборе способа оплаты товара.
`setAddress(address: string): void` - хранение данных адреса покупателя.
`setEmail(email: string): void` - хранение данных почты покупателя.
`setPhone(phone: string): void` - хранение данных номера телефона покупателя.
`getData(): IBuyer` - получение всех данных покупателя.
`clearPayment(): void` - очистка данных покупателя о выборе способа оплаты заказа.
`clearAddress(): void` - очистка данных адреса покупателя.
`clearEmail(): void` - очистка данных почты покупателя.
`clearPhone(): void` - очистка данных номера телефона покупателя.
`validateForm(): { [key: string]: string }` - валидация данных

#### Слой Представления

### Класс Gallery
Класс компонента для управления списком товара интернет-магазина. Реализует интерфейс IGallery и наследует функционал от класса Component. Предназначен для динамического отображения и обновления коллекции элементов в контейнере.

Конструктор: 
`container: HTMLElement` - контейнер для галереи.

`container: HTMLElement` - контейнер, содержащий структуру шапки.
`events: IEvents` - экземпляр системы событий для обработки взаимодействия.

Поля класса:
`basketButton: HTMLElement` - контейнер (HTML-элемент) кнопки корзины, при клике на которую открывается корзина.
`counterElement: HTMLElement` - контейнер (HTML-элемент) для отображения счетчика в корзине.

Методы класса:
`set сounter(value: number)` - устанавливает значение счетчика товаров в корзине.

### Класс Modal
Класс компонента для отображения модальных окон. Реализует интерфейс IModal и наследует функционал от класса Component. Предназначен для открытия, закрытия и управления содержимым модальных окон.

Конструктор: 
`container: HTMLElement` - контейнер, содержащий структуру модального окна.
`events: IEvents` - экземпляр системы событий для обработки взаимодействия.

Поля класса:
`closeButton: HTMLButtonElement` - контейнер кнопки закрытия модального окна.

Методы класса:
`set content(content: HTMLElement):` - устанавливает содержимое модального окна. Полностью заменяет предыдущее содержимое используя replaceChildren().

#### Класс Contact
Класс формы ввода контактных данных. Наследует от абстрактного класса Form и реализует функциональность для ввода email и телефона покупателя при оформлении заказа.

Конструктор:
`container: HTMLElement` - контейнер формы контактов.
`events: IEvents` - экземпляр системы событий для обработки взаимодействия.

Поля класса:
`inputEmail: HTMLInputElement` - контейнер поля ввода почты.
`inputPhone: HTMLInputElement` - контейнер поля ввода телефона.

Методы класса:
`set email(value: string)` - устанавливает значение в поле ввода email.
`set phone(value: string)` - устанавливает значение в поле ввода телефона.
